Constraint-aware design decisions

01 — Theme JSON ships the design system
Color tokens, type defaults, axis formatting exported as a single JSON. One source of truth for designers and BI developers.
02 — Bookmarks drive tab logic
Performance / Allocation / Risk tabs on Client Detail are Power BI bookmarks, not separate pages. Match BI's strengths, don't fight them.
03 — Field parameters for time horizon
1M / 3M / YTD / 1Y / 3Y toggles are field parameters, not separate visuals. Performant on large datasets.
04 — Drillthrough on Client Detail
Any client reference anywhere drills into the client page with context preserved.
05 — RLS enforced at the data layer
Row-Level Security means Jameson sees only her 45 clients. Security is built into the model, not the UI.
Power BI is the medium. The design respects what it can render natively, where it shines, and where it constrains — and works within those edges rather than fighting them.

Engineering Collaboration

  • Standalone components with signal-based inputs (Angular 17+)
  • Storybook coverage for every component — living documentation
  • TypeScript strict mode, signal inputs typed
  • Theme tokens shared between SCSS and TypeScript
  • Lazy-loaded chart behaviors for large datasets
  • Component states documented for every interaction

Accessibility & Performance

  • WCAG AAA contrast across all financial data
  • Tabular numerals enforced for monetary values
  • Color never the only signal (status uses color + icon + text)
  • Screen-reader friendly data tables with semantic markup
  • Colorblind-safe palettes for variance indicators
  • Component-level lazy loading for sub-second initial render
Each component documents both its Angular implementation and its Power BI mapping. The design system serves two stacks from one source of truth.
